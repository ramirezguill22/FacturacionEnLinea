Comandos para ejecutar el proyecto localmente

1. Instalar Node.js LTS en Windows si aun no esta instalado:
   winget install OpenJS.NodeJS.LTS

2. Cerrar y volver a abrir la terminal de VS Code.

3. Verificar que Node.js y npm esten disponibles:
   node -v
   npm -v

4. Entrar a la carpeta raiz del proyecto:
   cd c:\Users\guill\Documents\Portal_Facturacion_Online

5. frontend local:
   npm install

6. npm run dev

   --$env:Path = 'C:\Program Files\nodejs;' + $env:Path; npm.cmd run dev

7. Probar el frontend en el navegador:
   http://localhost:3000

8. Entrar a la carpeta del backend:
   cd c:\Users\guill\Documents\Portal_Facturacion_Online\backend

9. backend:
   npm install

10. backend local:
    npm run dev

    -- $env:Path = 'C:\Program Files\nodejs;' + $env:Path; npm.cmd run dev

11. Probar el endpoint del backend:
    POST http://localhost:8080/api/tickets/validar

    Body JSON:
    {
      "numeroTicket": "TICKET-PRUEBA-0000001"
    }

12. Comando PowerShell para probar el endpoint:
    Invoke-RestMethod -Method Post -Uri 'http://localhost:8080/api/tickets/validar' -ContentType 'application/json' -Body '{"numeroTicket":"TICKET-PRUEBA-0000001"}'

Resultado esperado del frontend:
Facturacion en linea

Resultado esperado del backend con configuracion real valida:
Respuesta JSON devuelta por NetSuite y propagada por el backend.

=====================================
BLOQUE FRONTEND CONECTADO AL BACKEND
=====================================

Flujo actual:
- /iniciar-facturacion envia el ticket al proxy interno de Next.js
- el proxy interno esta en /api/tickets/validar
- el proxy reenvia la peticion al backend real
- /resultado-validacion muestra el resultado real usando query params

Endpoint interno del frontend:
POST http://localhost:3000/api/tickets/validar

Configuracion opcional para el proxy:
BACKEND_API_BASE_URL=http://localhost:8080

Si no se define BACKEND_API_BASE_URL, el frontend usa por defecto http://localhost:8080.

========================================
BLOQUE ESTRUCTURA FRONTEND HACIA BACKEND
========================================

Patron oficial del proyecto:
- el frontend no debe construir URLs completas del backend en cada pantalla
- el frontend usa rutas internas de Next.js como fachada
- las rutas internas llaman al backend real usando una utilidad compartida

Archivos base creados para este patron:
- app/lib/backend/config.ts
- app/lib/backend/routes.ts
- app/lib/backend/client.ts

Responsabilidad de cada archivo:
- config.ts: obtiene la base URL del backend desde BACKEND_API_BASE_URL
- routes.ts: concentra las rutas funcionales del backend
- client.ts: reenvia peticiones JSON al backend y regresa una respuesta uniforme

Ruta interna actual conectada al patron:
- app/api/tickets/validar/route.ts

Variable de entorno del frontend:
- BACKEND_API_BASE_URL

Archivo de referencia:
- .env.example en la raiz del proyecto

Valor local recomendado:
BACKEND_API_BASE_URL=http://localhost:8080

Ejemplo de crecimiento futuro:
- backendRoutes.tickets.validar
- backendRoutes.tickets.elegibilidad
- backendRoutes.facturacion.datosFiscales
- backendRoutes.facturacion.confirmar

Regla recomendada:
Cada nuevo endpoint del portal debe seguir este patron antes de ser consumido por la UI.
